## 断电、备份和损坏恢复

测试在 2.5 秒高频写入期间直接强杀后端进程，共有 328 次写入收到成功响应。重启后第 328 次写入仍存在，SQLite `quick_check=ok`，日志模式为 `wal`。

随后执行：

1. 通过 API 创建在线一致性备份并下载，下载文件完整性检查通过。
2. 写入一个未备份的新值，强杀后端，然后覆盖破坏主数据库头部。
3. 重启后端，程序识别 `file is not a database`，隔离损坏文件并从最新有效备份恢复。
4. 修改恢复后的数据，再通过 API 手工恢复指定备份；恢复前回滚备份自动生成。
5. 通过桌面端使用的本机令牌关机接口退出，47ms 内完成退出备份和数据库关闭，进程退出码为 0。

自动化结果：`output/power-recovery-20260722T120901665Z-13508/result.json`（本机测试产物，不提交 Git）。

## 现场边界

软件能处理进程异常退出、数据库文件损坏和 Windows 登录后的自动启动。市电恢复后主机是否自行开机由 BIOS/主板决定；Windows 图形界面是否进入展示账号由系统登录策略决定。正式交付需开启 BIOS 来电自动开机、配置专用账号自动登录，并依据客户最终 PLC 型号和点表做现场验收。
