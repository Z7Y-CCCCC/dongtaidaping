# 引入“车间”层级的重构计划与系统使用指南

为了满足“车间-产线-炉子”的真实工业物理拓扑，需要在当前系统架构中插入一层“车间（Workshop）”。

## User Review Required

> [!WARNING]
> **数据库结构变更**：我们需要给数据库新增 `workshops` 表，并为现有的 `lines` 表增加 `workshop_id` 关联。我们会编写迁移代码来确保旧有产线自动归属到一个“默认车间”下，不会丢失数据。

> [!IMPORTANT]
> **大屏下钻视角的确认**：
> 目前大屏的视角是三级：全局视角 -> 产线视角 -> 单机视角。
> 加入车间后，变成了四级概念：工厂 -> 车间 -> 产线 -> 炉子。
> **在 3D 视角交互上，有两种方案，请您选择（后续我按您的选择执行）：**
> - **方案1（推荐）**：大屏仍保持 3 级运镜逻辑。在“全局视角”下看到的是整个工厂的所有设备（按车间分块布局），点击某个产线区域进入“产线视角”，再点击设备进入“单机视角”。“车间”作为一个布局的分组容器。
> - **方案2**：增加为 4 级运镜。全局 -> 点击车间 -> 车间视角 -> 点击产线 -> 产线视角 -> 单机视角。

---

## Proposed Changes

### 1. 后端数据结构层

#### [MODIFY] `backend/db/database.js`
- 新增 `workshops` (车间) 数据表
- 为 `lines` (产线) 数据表追加 `workshop_id` 字段
- 修改默认初始化的种子数据（创建 A 车间、B 车间，将产线分配给它们）
- 编写数据库迁移逻辑：如果表存在但缺少 `workshop_id` 字段，则自动 Alter Table 并关联默认车间。

#### [NEW] `backend/routes/workshops.js`
- 提供车间对象的增删改查 (CRUD) 接口。

#### [MODIFY] `backend/server.js`
- 挂载 `/api/workshops` 路由。

#### [MODIFY] `backend/routes/config.js`
- 修改聚合接口逻辑，输出的数据结构调整为：`workshops` -> 嵌套 `lines` -> 嵌套 `devices` -> 嵌套 `dataPoints`。

---

### 2. 前端配置驱动层与 UI

#### [MODIFY] `frontend/src/config/factoryConfig.js`
- 适配新的四级数据结构解析，暴露 `workshops` 对象。

#### [MODIFY] `frontend/src/views/AdminPanel.vue`
- 在“工厂拓扑与配置”页签中，左侧的目录树由原来的【产线-设备】升级为【车间-产线-设备】三层结构。
- 增加“添加车间”、“编辑车间”、“删除车间”的管理面板。
- 添加设备时，级联选择“车间 -> 产线”。

#### [MODIFY] `frontend/src/App.vue` & `frontend/src/three/SceneManager.js`
- 重新计算 3D 场景中的排布算法：以“车间”为单位划分区域，车间内部排列“产线”，产线内部排列“炉子”。
- （待定）根据您上方选择的“运镜方案”，调整对应层级的镜头焦点逻辑。

## 系统的完整使用配置流程 (当前已实现)

在实施上述改动之前，为您梳理数字孪生平台现有的使用与配置链路：

1. **环境准备与启动**
   - 启动后端服务 (`npm start`) 和 前端服务 (`npm run dev`)。
   - 访问 `http://localhost:5173/admin` 进入后台管理面板。

2. **绘制物理拓扑 (添加产线和设备)**
   - 在后台管理的“工厂拓扑与配置”选项卡，先新建产线，再为产线添加多台多用炉设备。
   - 设置设备的型号、位置参数（可以通过修改 X、Z 坐标决定 3D 界面上炉子的排列方式）。

3. **PLC 通信配置 (核心链路)**
   - 在后台切换到“连接设置”选项卡。
   - **步骤A**：选择您的数据流模式（强烈推荐目前已实现的 **方案B：Node.js 直连 S7 PLC**）。
   - **步骤B**：填写您的真实 PLC IP 地址、端口(102)、机架号(0)、槽号(1)。
   - **步骤C**：保存设置后，系统会自动调用接口重启数据引擎，开始尝试连接 PLC。页面最上方的数据引擎状态卡片会显示“🟢 通信正常”。

4. **点位映射 (数据对齐)**
   - 切换到后台的“点位映射”选项卡。
   - 在左侧选择一台多用炉。系统列出了必须采集的核心变量（如温度、碳势、各电机状态）。
   - 在 `PLC地址 (例如 DB1,REAL0)` 一列，根据您的博图/PLC 变量表，填写真实的 S7 寄存器地址。
   - 配置好一台炉子后，可以使用上方的**“同步到同产线设备”**按钮，自动将相同的 DB 块相对偏移地址结构复制给其它炉子（之后只需微调不同炉子的 DB 块号）。

5. **展示与监控**
   - 访问大屏本体 `http://localhost:5173/`。
   - 大屏前端通过 WebSocket 连接到后端，不再需要配置任何 MQTT，只要后端的 PLC 引擎连通，大屏上每个炉子的温度、状态灯、模型特效（门、推拉链）就会直接实时跑起来。
